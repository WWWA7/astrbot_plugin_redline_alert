# 红线预警

一个 AstrBot 插件，用于注册名为 **“红线预警”** 的 LLM 工具。

当模型判断当前对话已经涉及明显高风险内容，例如政治红线、煽动颠覆、分裂国家、严重违规引导等内容时，可以主动调用该工具。插件会把：

- 当前对话总结
- 危险点说明
- 风险等级
- 建议处理方式（可选）

发送到插件设置中配置好的群聊或私聊用户，用于人工接收预警。

---

## 功能特性

- 注册一个 LLM 工具：`redline_alert`（中文含义：红线预警）
- 支持将预警发送到指定群聊
- 支持将预警发送到指定私聊用户
- 支持同一会话冷却时间，避免重复刷屏
- 支持附带发送者 ID、群聊 ID 等来源信息
- 支持限制摘要、危险点、建议文本长度

## 配置项说明

### enabled
是否启用插件。

### platform
发送平台，当前建议使用：

```text
aiocqhttp
```

### notify_user_ids
接收预警的用户 ID 列表，例如 QQ 号。

示例：

```json
["12345678", "87654321"]
```

### notify_group_ids
接收预警的群聊 ID 列表。

示例：

```json
["123456789", "987654321"]
```

### cooldown_seconds
同一会话重复触发预警时的冷却时间，默认 300 秒。

### alert_title
发送出去的预警消息标题，默认：

```text
【红线预警】
```

### include_origin
是否在预警消息中附带来源信息，例如：

- 发送者 ID
- 群聊 ID
- 是否为私聊

### max_summary_length
对话总结最大长度。

### max_danger_length
危险点说明最大长度。

### max_suggestion_length
建议处理说明最大长度。

## 工具说明

插件注册的实际工具名为：

```text
redline_alert
```

该工具的中文含义是“红线预警”。

### 工具用途

当模型明确判断当前对话具有较高风险时调用，而不是用于普通争议、日常敏感词、学术讨论或无害上下文。

### 工具参数

#### risk_level
风险等级，例如：

- `low`
- `medium`
- `high`
- `critical`

#### danger_points
危险点总结，说明哪里危险。

#### conversation_summary
当前对话总结，便于管理员快速了解上下文。

#### suggested_response
建议采取的处理方式，例如：

- 拒绝继续该话题
- 引导更换话题
- 提示内容违规

## 预警消息示例

```text
【红线预警】
平台：aiocqhttp
发送者 ID：12345678
群聊 ID：987654321
风险等级：high
危险点：对话涉及现实政治红线，并带有明显煽动和规避表达
对话总结：用户连续要求生成涉及政治红线内容，并试图诱导模型继续扩展敏感叙述
建议处理：停止继续该话题，明确拒绝，并引导对方更换安全话题
```

---

## 工作机制

当 LLM 调用 `redline_alert` 工具后，插件会：

1. 读取工具参数
2. 生成预警消息
3. 向配置中的用户或群聊发送消息
4. 记录当前会话最后一次预警时间
5. 在冷却时间内阻止重复发送

---

## 注意事项

1. 当前主动发送使用的是 `StarTools.send_message_by_id(...)`
2. 目前实现优先适配 `aiocqhttp`
3. 如果未配置任何接收目标，工具会返回提示，但不会真正发送预警
4. 如果目标 ID 错误或平台未连接，发送会失败并在返回结果中体现
5. 该插件只负责“上报预警”，不会自动封禁、踢人或执行额外处罚逻辑

---

## 适用场景

适合希望在模型遇到高风险对话时，第一时间把摘要上报到：

- 管理员私聊
- 审核群
- 风控群

以便人工及时关注。
